# Flask构建后端接口

# 说明

我为什么一直想将flask和Vue结合起来？因为前端项目中，Vue的组件切换很给力，不管是移动端还是PC端，都可以做出非常丰富的内容，而且可以进行丰富的用户交互，这给了我很多的创意空间。

而flask的前端表现不是很好，需要给出明显的路由才能进行跳转（通过<a></a>标签来进行跳转），不符合SPA的规则；而且用户交互也不是很好，比如说我要操作表格，增删改查就需要很多JS代码，这就太麻烦了。

我也尝试过在flask项目的html文件中引入Vue，是可以使用，但是很明显，不能使用.vue的文件，也就是不能使用vue组件，这样就相当于是一种简化版的JS代码，没起什么作用，所以我在网上搜索了一下，是否可以将flask和Vue结合起来，这样就免得我另外学习node.js了（虽然最后免不了还是要学），但是我觉得flask构建简单网站是有优势的：简单、快速。这样对于我来说的话就很有用。

# 在app.py文件中

from flask import Flask, render_template

app = Flask(__name__)

# 根目录返回index.html
@app.route('/')
def hello_world():
    return render_template(
        'index.html',
    )

# /main 根目录下的main，返回main.html
@app.route('/main')
def hello_world2():
    return render_template(
        'main.html'
    )

if __name__ == '__main__':
    app.run(debug=True)

可以看到，在app.py文件中，定义了不同的路由地址，然后返回不同的html页面，这样就做到了页面的切换。但并不能做到Vue路由的切换。那么flask和Vue分别作为后端和前端，如何分离呢？

# 2.前后端分离项目如何启动

  1. 下载此项目。

  2. 命令行中打开项目的back_end文件夹，安装requirements.txt文件中的第三方包，运行app.py文件。

    $ cd back_end
$ python3.9 -m venv env
(env)$ pip install -r requirements.txt
(env)$ python app.py

    后端接口为： http://localhost:8080

    提示：若想在手机浏览器中浏览项目，需将app.py文件中的localhost地址改为本地的IPv4地址。

  3. 在另外一个命令行中打开front_end文件夹，安装第三方包，并运行。

    $ cd front_end
$ npm install
$ npm run serve

    前端网址为： http://localhost:8999。由于本项目是移动端项目，在桌面端打开时，打开调试窗口的手机窗口查看效果更好。

    提示：若想在手机浏览器中浏览项目，需将vue.config.js和src/service/http.ts文件中的localhost地址改为本地的IPv4地址。

# 3.flask端如何做

# 3.1 安装Flask-Cors扩展

pip install Flask-Cors，flask-cors的官网：https://flask-cors.readthedocs.io/en/latest/。

为什么要安装Flask-Cors扩展？

是为了发出跨域请求。比如，来自不同协议，IP地址，域名或端口的请求——而Flask-CORS可以帮我们处理这些。

需要注意的是，虽然上面的设置允许所有路由上的跨域请求（来自任何域、协议或端口）。但在生产环境中，您应该只允许来自托管前端应用程序的域的跨域请求。有关此问题的更多信息，请参阅Flask-CORS文档 (opens new window)。

# 3.2 创建app.py文件

使用pycharm创建flask项目时，会生成很多文件夹，其实真正需要的文件只有一个，那就是app.py文件，所以可以直接在flask项目的文件夹下，创建一个名为app.py的文件，里面放如下代码：

from flask import Flask, jsonify
from flask_cors import CORS


# 可以热更新
DEBUG = True

# 实例化 app
app = Flask(__name__)
app.config.from_object(__name__)

# 开通 CORS
CORS(app, resources={r'/*': {'origins': '*'}})


# 完整性检查 route，这里路由的地址是 http://localhost:5000/ping ，我的localhost设置的是127.0.0.1，所以地址就是http://127.0.0.1:5000/ping
@app.route('/ping', methods=['GET'])
def ping_pong():
    return jsonify('pong!')


if __name__ == '__main__':
    app.run()

上述代码有两个地方比较重要：

  1. @app.route('/ping', methods=['GET'])，这句代码规定了前端请求数据的地址为：http://127.0.0.1:5000/ping，我还可以设置很多路由，通过路由地址的不同，来返回不同的数据。methods=['GET']，用来指定这个地址的方法，可以用来对后端请求数据。
  2. jsonify('pong!')，为什么返回值要写成json格式？因为前端接收的数据一般用对象来接收，写成json格式是为了方便前端处理数据。当然，这也不可能硬性要求是json数据，可以在明确数据类型的情况下，写成别的类型的数据。自己或者对方知道怎么处理就行了。

# 5.数据接口

使用axios来获取数据，在ping.vue文件中，修改<script>里面的代码：

<script>
// 引入axios
import axios from 'axios';

export default {
  name: 'Ping',
  data() {
    return {
      msg: '',
    };
  },
  created() {
    this.getMessage();
  },
  methods: {
    getMessage() {
      const path = 'http://localhost:5000/ping';//这里的/ping应该只是为了和后端保持一致，不是硬性要求
      axios.get(path)    //axios的用法没有变
        .then((res) => {
          this.msg = res.data;
        })
        .catch((error) => {
          // eslint-disable-next-line
          console.error(error);
        });
    },
  },
};
</script>

可以在全局的js文件中，设置axios的请求地址前缀，这样就方便我们，不用一直写完整的路由地址：

import axios from "axios";
Vue.prototype.$http = axios.create({
  baseURL: "http://localhost:8080", //在全局设置 axios 的请求根路径，这样，在发起请求的时候，就可以不写根路径了，具体根路径还要看你的设置
});

# 6.app.py文件怎么设计

如果我们请求的数据很多，那岂不是要在app.py文件中写很多代码，那么这个文件用起来就不是很方便了，很难进行维护。可以这样做，充分发挥Python模块化的精髓，把每一个重要模块写成.py文件，然后在app.py文件中导入使用即可。

数据也可能有很多，那么我可以创建很多.json文件，使用Python的json库，来处理这些文件。可能还要涉及到json与其余数据类型之间的转换。

# 7.为什么数据传输以json为主？

JSON(JavaScript Object Notation) 是一种轻量级的数据交换格式。它基于ECMAScript的一个子集。 JSON采用完全独立于语言的文本格式，但是也使用了类似于C语言家族的习惯(包括C、C++、Java、JavaScript、Perl、Python等)。这些特性使JSON成为理想的数据交换语言。易于人阅读和编写，同时也易于机器解析和生成(一般用于提升网络传输速率)。

pandas本身就为我们考虑到了转换为json格式的问题。

# 8.什么叫RESTful API

rest的全称是Representational State Transfer，2000年由Roy Fielding发表在他的博士论文中，首次提出了Restful的概念。Restful 做为一种接口的设计规则，基于HTTP协议 ，可以使用XML格式 或JSON格式传输数据。对于使用restful 这套规则定义开发的接口，我们称之为Restful API。 关于Restful的具体表现形式，简单可以用一句话概括：URL定位资源，使用HTTP请求（GET、POST、PUT、DELETE等）对资源进行操作。

HTTP 动词

  • GET: 获取资源
  • POST：创建或更新资源
  • PUT: 创建或更新资源
  • DELETE：删除资源
  • HEAD：获取资源的元数据，不常用
  • OPTIONS：获取信息，关于资源的哪些属性是客户端可以改变的， 不常用

# 9.GET和POST详解

get和post 1、get请求： 使用场景：如果只对服务器获取数据，并没有对服务器产生任何影响，那么这时候使用get请求 传参：get请求传参是放在url中，并且是通过？的形式来指定key和value的 2、post请求： 使用场景：如果要对服务器产生影响，那么使用post请求 传参：post请求传参不是放在URL中，是通过form data 的形式发送给服务器的 get 请求是通过flask.request.args来获取 post请求是通过flask.request.form来获取 post请求在模板中要注意几点: *input 标签中，要写name来表示这个value的key,方便后台获取 *在写form表单的时候，要指定method=‘post’,并且要指定action='/login/'

  • GET

对于前端，axios.get()就是从后端获取数据。

对于后端，需要在路由中写入methods=['GET']，就表示前端从后端获取数据，一般不需要使用request.method = 'GET'来判断方法，只需要在函数中return数据即可。以下是示例：

  • POST

对于前端，axios.post()就是向后端发送数据。

对于后端，需要在路由中写入methods=['GET','POST']，就表示前端向后端发送数据，要使用request.method = 'POST'来判断方法，并且在函数中return最新的数据。以下是示例：

前端部分：

<script>
import axios from 'axios';

export default {
  data() {
    return {
      books: [],
      addBookForm: {
        title: '',
        author: '',
        read: [],
      },
    };
  },
  created() {
    this.getBooks();
  },
  methods: {
    // 获取数据的方法
    getBooks() {
      const path = 'http://localhost:5000/books';
      axios.get(path)
        .then((res) => {
          this.books = res.data.books;
        })
        .catch((error) => {
          // eslint-disable-next-line
          console.error(error);
        });
    },
      
    // 添加数据的方法
    addBook(payload) {
      const path = 'http://localhost:5000/books';
      axios.post(path, payload)    // 第一个参数是地址，第二个参数是数据
        .then(() => {
          this.getBooks();         // 在添加数据之后，重新获取数据，达到更新数据的目的
        })
        .catch((error) => {
          // eslint-disable-next-line
          console.log(error);
          this.getBooks();
        });
    },
      
    // 初始化添加数据功能的表格为空
    initForm() {
      this.addBookForm.title = '';
      this.addBookForm.author = '';
      this.addBookForm.read = [];
    },
      
    // 这里是调用添加数据这个方法
    onSubmit(evt) {
      evt.preventDefault();
      this.$refs.addBookModal.hide();
      let read = false;
      if (this.addBookForm.read[0]) read = true;
      const payload = {
        title: this.addBookForm.title,
        author: this.addBookForm.author,
        read, // property shorthand
      };
      this.addBook(payload);
      this.initForm();
    },
      
    // 重置按钮
    onReset(evt) {
      evt.preventDefault();
      this.$refs.addBookModal.hide();
      this.initForm();
    },
  },
};
</script>

后端部分：

from flask import Flask, jsonify, request

@app.route('/books', methods=['GET', 'POST'])
def all_books():
    response_object = {'status': 'success'}
    if request.method == 'POST':
        post_data = request.get_json()    #这里使用的是request的get_json()方法
        BOOKS.append({
            'title': post_data.get('title'),
            'author': post_data.get('author'),
            'read': post_data.get('read')
        })
        response_object['message'] = 'Book added!'
    else:
        response_object['books'] = BOOKS
    return jsonify(response_object)
